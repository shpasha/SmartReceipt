## 1. Чипы дробей при фокусе на ввод количества

**Поведение:**
- При фокусе на input в `UnitsStepper` — под позицией скрыть чипы участников и показать
  **чипы дробей** в их месте (не добавлять, а заменять, чтобы не разрастать карточку).
- Дроби зависят от количества участников в комнате: при N людях → `½, ⅓, ¼ ... 1/N`.
  Например 4 человека → `½, ⅓, ¼`. Скип `1/1`.
- Тап на чип → ставит значение в инпут, инпут остаётся фокусированным или коммитит.
- На blur — чипы исчезают, возвращаются чипы участников.

## 5. Уйти с Docker на нативный Node + systemd

**Зачем:** docker build ~1.5 мин на каждый деплой. Для одного Node на одном VPS
Docker — оверхед без выгод. Деплой ускорится до 10-30с.

**Что готово в репо:**
- `scripts/deploy.sh` — rsync + npm ci + npm run build + systemctl restart.
- `scripts/smartreceipt.service` — systemd-юнит, запускает `.next/standalone/server.js`.

**Одноразово на сервере (root@<PROD_HOST>):**
1. Поставить Node 20: `curl -fsSL https://deb.nodesource.com/setup_20.x | bash - && apt install -y nodejs`.
2. Положить env-файл `<ENV_DIR>/env` (ANTHROPIC_API_KEY, HTTPS_PROXY и т.д. —
   взять из текущего docker `--env-file` или контейнера).
3. Скопировать юнит: `scp scripts/smartreceipt.service root@<PROD_HOST>:/etc/systemd/system/`
   (или `cp <DEPLOY_DIR>/scripts/smartreceipt.service /etc/systemd/system/`).
4. `systemctl daemon-reload && systemctl enable smartreceipt`.
5. Прогнать первый `npm run deploy` локально — он сделает rsync, build и `systemctl restart`.
6. Когда сервис поднялся: `docker rm -f smartreceipt` (остановить и снести старый контейнер).
7. Логи: `journalctl -u smartreceipt -f`.

Nginx уже проксит на `127.0.0.1:3001` — менять не надо.

Volume под картинки (#3) станет просто `<STATE_DIR>/uploads`.
