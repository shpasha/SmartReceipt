## 1. Реконнект участника без grace-period · ✅ сделано, надо проверить

**Проблема:** на Android при сворачивании/разворачивании браузер обрывает SSE; после
возврата у участника не кликабельны кнопки. Текущий `PRESENCE_GRACE_MS = 30_000` лишний.

**Поведение:**
- Grace-period убрать полностью.
- Если у участника **0 выборов** и SSE-коннект пропал → **сразу удалить** запись из
  комнаты (сейчас — после 30с). Другие перестают его видеть моментально.
- Если у участника **есть выборы** — оставить запись (нельзя терять данные при offline).
- При reconnect (браузер вернулся в фокус, SSE открылся снова) клиент имеет в
  `localStorage` старый `me {id, name}`. Но id на сервере уже мог быть удалён.
- Клиент проверяет: есть ли мой `me.id` в свежем `room.participants`?
  - Да → ничего делать не нужно, продолжаем работу.
  - Нет → автоматически переподключаемся под тем же именем (`POST /join { name }`):
    - Если `200` → сервер создал нового участника (новый id), сохраняем в
      `localStorage`, продолжаем.
    - Если `409 name_taken` (имя уже занято кем-то) → чистим `localStorage[me]` и
      ведём на экран ввода имени (текущий join-флоу).
- Кнопки «не кликабельны после reconnect» — это и есть симптом этого бага: id у
  клиента есть, на сервере нет, все запросы select/leave валятся.

## 2. Чипы дробей при фокусе на ввод количества

**Поведение:**
- При фокусе на input в `UnitsStepper` — под позицией скрыть чипы участников и показать
  **чипы дробей** в их месте (не добавлять, а заменять, чтобы не разрастать карточку).
- Дроби зависят от количества участников в комнате: при N людях → `½, ⅓, ¼ ... 1/N`.
  Например 4 человека → `½, ⅓, ¼`. Скип `1/1`.
- Тап на чип → ставит значение в инпут, инпут остаётся фокусированным или коммитит.
- На blur — чипы исчезают, возвращаются чипы участников.
